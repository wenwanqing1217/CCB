# 校园盲盒即时配送平台

基于微信小程序云开发的**C2C+B2C**双模式校园盲盒交易与即时配送平台，服务于武汉生物工程学院在校师生及周边商家。

## 项目简介

### 项目定位
校园场景下的**闲置物品交易** + **即时配送服务**平台，支持**个人卖家（C2C）**和**认证商家（B2C）**两种模式。

### 解决的问题
| 问题 | 痛点 |
|------|------|
| 个人闲置 | 占地方、难出手、信息不对称 |
| 认证商家 | 校园渠道推广难、配送成本高 |
| 学生买家 | 寻找低价好物、即时配送上门、趣味购物体验 |
| 学生骑手 | 勤工俭学、灵活自由、顺路赚钱 |

### 核心价值
- **C2C模式**：个人发布闲置盲盒，快速变现
- **B2C模式**：认证商家发布商品，校园渠道销售
- **即时配送**：学生骑手抢单配送，解决最后一公里

## 技术栈

- **前端**：微信小程序原生框架 (WXML/WXSS/JS)
- **后端**：微信云开发（云函数、云数据库、云存储）
- **数据库**：云开发数据库（MongoDB 兼容）
- **实时通信**：云数据库实时监听
- **推送服务**：微信订阅消息

## 核心功能模块

### 🛍️ 买家端
- 盲盒浏览、搜索、分类筛选
- 热门推荐 + 个性化混合推荐
- 下单购买、订单追踪
- 骑手位置实时查看
- 积分系统、收藏管理

### 🏪 卖家端（C2C + B2C）
- **个人卖家**：发布闲置盲盒、商品管理、订单查看
- **认证商家**：商家入驻申请、批量发布商品、库存管理、销售统计

### 🏃 骑手端
- 附近订单查看、抢单
- 实时位置上报
- 配送状态更新
- 配送统计、收益管理

### 🛠️ 管理端
- 用户管理、骑手管理
- 商品审核、订单监控
- 商家入驻审核
- 数据统计（用户增长、订单量、交易额等）

## 数据库设计

核心集合：
- `users` - 用户信息
- `boxes` - 盲盒商品
- `orders` - 订单记录
- `deliveries` - 配送记录
- `donations` - 捐赠记录
- `userActions` - 用户行为日志
- `riders` - 骑手信息
- `notifications` - 通知消息

## 项目亮点

1. **混合推荐算法** - 内容推荐 + 协同过滤，60:40权重混合，提升推荐准确率
2. **订单状态实时同步** - 云数据库实时监听，订单状态变更即时推送至用户端
3. **骑手位置实时跟踪** - 高德地图SDK集成，配送路线可视化
4. **原子性并发控制** - 抢单使用原子更新，防止超卖问题
5. **性能监控体系** - 多维度监控（FPS、内存、网络、API耗时）
6. **智能缓存策略** - LRU + 多级过期时间，自动清理
