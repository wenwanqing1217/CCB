# 数据库索引设计

## 概述

合理的索引设计能大幅提升查询性能。本文档针对校园盲盒平台的核心业务场景，制定索引策略。

---

## 集合索引设计

### 1. 用户集合（users）

```javascript
db.users.createIndex({ "openid": 1 }, { unique: true })
db.users.createIndex({ "role": 1 })
db.users.createIndex({ "campus": 1, "dormitory": 1 })
db.users.createIndex({ "createdAt": -1 })
```

| 索引字段 | 类型 | 唯一 | 用途 |
|---------|------|-----|------|
| openid | 1 | ✓ | 用户唯一标识，主键查询 |
| role | 1 | - | 按角色筛选（学生/骑手/商家） |
| campus + dormitory | 1 | - | 校内位置查询 |
| createdAt | -1 | - | 按注册时间排序 |

---

### 2. 盲盒集合（boxes）

```javascript
// 主键索引
db.boxes.createIndex({ "_id": 1 }, { unique: true })

// 查询索引
db.boxes.createIndex({ "status": 1, "createdAt": -1 })
db.boxes.createIndex({ "status": 1, "type": 1, "createdAt": -1 })
db.boxes.createIndex({ "status": 1, "campus": 1, "createdAt": -1 })
db.boxes.createIndex({ "sellerOpenid": 1, "createdAt": -1 })
db.boxes.createIndex({ "category": 1, "price": 1 })

// 全文搜索索引
db.boxes.createIndex({ "title": "text", "desc": "text" })
```

| 索引字段 | 类型 | 用途 |
|---------|------|------|
| status + createdAt | 复合 | 首页列表查询（按状态+时间） |
| status + type + createdAt | 复合 | 分类筛选列表 |
| status + campus + createdAt | 复合 | 校区筛选列表 |
| sellerOpenid + createdAt | 复合 | 我的发布查询 |
| category + price | 复合 | 价格区间筛选 |
| title + desc | 全文 | 关键词搜索 |

## 复合索引设计原则

### 1. 最左前缀匹配

```javascript
// 定义复合索引
db.boxes.createIndex({ "status": 1, "type": 1, "createdAt": -1 })

// 以下查询能使用索引
db.boxes.find({ "status": "available" })
db.boxes.find({ "status": "available", "type": "数码" })
db.boxes.find({ "status": "available", "type": "数码", "createdAt": { $gt: xxx } })

// 以下查询无法使用索引
db.boxes.find({ "type": "数码" })  // 跳过 status
```

### 2. 区分度高的字段放前面

```javascript
// 好的设计：status（区分度低）在前，createdAt（区分度高）在后
db.boxes.createIndex({ "status": 1, "createdAt": -1 })

// 查询时先过滤 status，再按时间排序
db.boxes.find({ "status": "available" }).sort({ "createdAt": -1 })
```

### 3. 等值查询字段在前，范围查询字段在后

```javascript
// 买家订单：等值(status)在前，排序(createdAt)在后
db.orders.createIndex({ "buyerOpenid": 1, "status": 1, "createdAt": -1 })
```

## 索引维护

### 1. 查看慢查询

```javascript
db.setProfilingLevel(1, { slowms: 100 })
db.system.profile.find({ millis: { $gt: 100 } }).sort({ ts: -1 })
```

### 2. 分析索引使用情况

```javascript
db.boxes.find({ "status": "available" }).explain("executionStats")
```

### 3. 索引重建（定期维护）

```javascript
db.boxes.reIndex()
```

### 4. 删除无用索引

```javascript
db.boxes.dropIndex("unused_index_name")
```

## 分页查询优化

### 游标分页（推荐）

```javascript
// 第一页
const firstPage = await db.collection('boxes')
  .where({ status: 'available' })
  .orderBy('createdAt', 'desc')
  .limit(20)
  .get()

const lastCursor = firstPage.data[firstPage.data.length - 1]._id

// 下一页
const nextPage = await db.collection('boxes')
  .where({
    status: 'available',
    _id: db.command.lt(lastCursor)
  })
  .orderBy('createdAt', 'desc')
  .limit(20)
  .get()
```

### Offset 分页（不推荐大偏移量）

```javascript
// 不推荐：skip(10000) 仍然会扫描前10000条数据
db.collection('boxes').skip(10000).limit(20)
```

## 注意事项

1. **索引不是越多越好**：每增加一个索引都会影响写入性能
2. **复合索引优于多个单字段索引**：节省空间，提高效率
3. **定期分析慢查询**：根据实际业务调整索引
4. **微信云开发限制**：每个集合最多10个索引
