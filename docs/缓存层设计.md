# 缓存层设计文档

## 一、缓存介质选型

| 介质 | 用途 | 原因 |
|------|------|------|
| Redis String | 用户Token、配置 | 简单高效 |
| Redis Hash | 用户信息、盲盒详情 | 结构清晰 |
| Redis List | 热门推荐、最近浏览 | 有序 |
| Redis Sorted Set | 排行榜 | 按分数排序 |
| Redis Geo | 骑手位置、附近订单 | 地理位置查询 |

## 二、缓存策略

### 1. 盲盒详情缓存

| 配置 | 值 |
|------|-----|
| Key | `box:detail:{box_id}` |
| 过期时间 | 30分钟 |
| 淘汰策略 | LRU |

```java
// 伪代码示例（Java）
public Box getBoxDetail(String boxId) {
    String key = "box:detail:" + boxId;
    
    // 1. 查缓存
    String cached = redis.opsForValue().get(key);
    if (cached != null) {
        return JSON.parseObject(cached, Box.class);
    }
    
    // 2. 查数据库
    Box box = boxDao.findById(boxId);
    if (box != null) {
        // 3. 写缓存
        redis.opsForValue().set(key, JSON.toJSONString(box), 30, TimeUnit.MINUTES);
    }
    
    return box;
}

public void updateBox(Box box) {
    // 1. 先更新数据库
    boxDao.update(box);
    
    // 2. 删缓存
    String key = "box:detail:" + box.getId();
    redis.delete(key);
}
```

### 2. 骑手位置缓存

| 配置 | 值 |
|------|-----|
| Key | `delivery:rider:location` |
| 过期时间 | 5秒 |
| 数据结构 | Redis Geo |

```go
// 伪代码示例（Go）
func UpdateRiderLocation(riderId string, lat, lng float64) {
    key := "delivery:rider:location"
    ctx := context.Background()
    
    // 更新位置
    _, err := rdb.GeoAdd(ctx, key, &redis.GeoLocation{
        Name:      riderId,
        Latitude:  lat,
        Longitude: lng,
    }).Result()
    
    if err != nil {
        log.Printf("更新位置失败: %v", err)
    }
    
    // 设置过期时间
    rdb.Expire(ctx, key, 5*time.Second)
}
```

### 3. 热门推荐缓存

| 配置 | 值 |
|------|-----|
| Key | `recommendation:hot` |
| 过期时间 | 5分钟 |
| 数据结构 | Redis List |

```python
# 伪代码示例（Python）
def refresh_hot_recommendation():
    # 计算热门推荐
    recommendations = calculate_hot()
    
    # 写入缓存
    redis.delete('recommendation:hot')
    for item in recommendations:
        redis.rpush('recommendation:hot', json.dumps(item))
    
    redis.expire('recommendation:hot', 300)  # 5分钟
```

## 三、缓存异常处理

### 缓存穿透

**问题：** 查询不存在的数据，缓存和数据库都没有，请求直接打到数据库

**解决方案：** 布隆过滤器

```java
// 初始化布隆过滤器
private static final BloomFilter<String> BLOOM_FILTER = BloomFilter.create(
    Funnels.stringFunnel(StandardCharsets.UTF_8),
    1000000,
    0.01
);

// 启动时将所有盲盒ID加入布隆过滤器
@PostConstruct
public void init() {
    List<String> boxIds = boxDao.findAllIds();
    for (String id : boxIds) {
        BLOOM_FILTER.put(id);
    }
}

// 查询时先过布隆过滤器
public Box getBoxDetail(String boxId) {
    if (!BLOOM_FILTER.mightContain(boxId)) {
        return null; // 布隆过滤器说没有，就真的没有
    }
    // ... 继续查缓存和数据库
}
```

### 缓存击穿

**问题：** 某个热点Key过期，大量请求同时打到数据库

**解决方案：** 互斥锁

```java
public Box getBoxDetail(String boxId) {
    String key = "box:detail:" + boxId;
    String lockKey = "lock:box:" + boxId;
    
    // 1. 查缓存
    String cached = redis.opsForValue().get(key);
    if (cached != null) {
        return JSON.parseObject(cached, Box.class);
    }
    
    // 2. 尝试获取锁
    Boolean locked = redis.opsForValue().setIfAbsent(lockKey, "1", 10, TimeUnit.SECONDS);
    if (locked) {
        try {
            // 3. 拿到锁后再次查缓存（双重检查）
            cached = redis.opsForValue().get(key);
            if (cached != null) {
                return JSON.parseObject(cached, Box.class);
            }
            
            // 4. 查数据库
            Box box = boxDao.findById(boxId);
            if (box != null) {
                redis.opsForValue().set(key, JSON.toJSONString(box), 30, TimeUnit.MINUTES);
            }
            
            return box;
        } finally {
            // 5. 释放锁
            redis.delete(lockKey);
        }
    } else {
        // 6. 没拿到锁，休眠一会重试
        try {
            Thread.sleep(100);
        } catch (InterruptedException e) {}
        return getBoxDetail(boxId);
    }
}
```

### 缓存雪崩

**问题：** 大量Key同时过期，请求雪崩到数据库

**解决方案：** 随机过期时间 + 降级

```java
public void setCache(String key, Object value) {
    // 基础过期时间 + 随机值（0-5分钟）
    int baseMinutes = 30;
    int randomSeconds = ThreadLocalRandom.current().nextInt(300);
    long expireSeconds = baseMinutes * 60 + randomSeconds;
    
    redis.opsForValue().set(key, JSON.toJSONString(value), expireSeconds, TimeUnit.SECONDS);
}
```

## 四、缓存监控

### 关键指标

| 指标 | 说明 | 告警阈值 |
|------|------|----------|
| 缓存命中率 | hit/(hit+miss) | < 90% |
| 缓存使用率 | 内存使用率 | > 80% |
| 大Key数量 | value > 10KB | > 100 |
| 慢查询 | 耗时 > 100ms | > 10次/分钟 |
