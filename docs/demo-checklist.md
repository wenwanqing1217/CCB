## 3. 演示技巧

### 展示顺序（推荐）
1. **首页 & 盲盒浏览**（展示 UI+数据加载）
2. **发布盲盒**（展示表单+云函数调用）
3. **下单 & 抢单**（展示核心业务逻辑）
4. **配送追踪**（展示实时功能）
5. **管理后台**（展示全栈能力）
6. **翻代码**（展示代码质量和架构）
7. **展示测试**（加分项）

### 话术要点
- 讲功能时顺带提技术实现（"这个列表用了虚拟列表优化"）
- 主动抛出难点（"这里最难的是抢单的并发控制，我用了原子更新..."）
- 控制节奏，不要在一个功能上卡太久
- 被问倒时诚实说"这个当时没考虑到，但如果再做我会..."

## 4. 常见翻车预防

| 风险 | 预防 |
|------|------|
| 云函数超时 | 提前测试，确保数据量不大 |
| 图片加载失败 | 使用本地占位图或确认图床可用 |
| 支付流程卡住 | 微信支付需要商户号，演示走模拟流程 |
| 数据库无数据 | 预置演示数据脚本 |
| 编译报错 | 演示前重新编译一次确认无报错 |
