## 二、Prometheus + Grafana（指标监控）

### 关键指标

| 指标类型 | 指标名 | 说明 | 告警阈值 |
|----------|--------|------|----------|
| **系统指标** | cpu_usage | CPU使用率 | > 80% |
| | memory_usage | 内存使用率 | > 85% |
| | disk_usage | 磁盘使用率 | > 90% |
| | network_in | 网络入流量 | - |
| | network_out | 网络出流量 | - |
| **应用指标** | http_requests_total | HTTP请求总数 | - |
| | http_request_duration_seconds | HTTP请求耗时 | P99 > 500ms |
| | http_requests_error_total | 错误请求数 | 错误率 > 1% |
| | jvm_memory_used_bytes | JVM内存使用 | - |
| | jvm_threads_live | JVM线程数 | - |
| | db_connection_pool_active | 数据库活跃连接数 | > 90% |
| | db_query_duration_seconds | SQL查询耗时 | P99 > 200ms |
| **中间件指标** | redis_connected_clients | Redis连接数 | - |
| | redis_memory_used_bytes | Redis内存使用 | > 80% |
| | mq_topic_messages | MQ消息数 | - |
| | mq_consumer_lag | 消费者积压 | > 10000 |
| **业务指标** | order_created_total | 订单创建数 | - |
| | user_active_total | 活跃用户数 | - |
| | delivery_success_rate | 配送成功率 | < 95% |

### Grafana 看板设计

**看板1：系统概览**
- CPU使用率
- 内存使用率
- 磁盘使用率
- 网络流量
- 服务存活状态

**看板2：应用性能**
- QPS/TPS
- 响应时间（P50/P95/P99）
- 错误率
- API TOP10 耗时

**看板3：数据库监控**
- 连接池使用率
- 慢查询数
- QPS
- 主从延迟

**看板4：中间件监控**
- Redis连接数
- Redis内存使用率
- MQ消息量
- MQ消费积压

**看板5：业务监控**
- 订单创建数
- 配送成功率
- 活跃用户数
- 交易额

### 告警规则

```yaml
groups:
  - name: system_alerts
    rules:
      - alert: HighCpuUsage
        expr: cpu_usage > 0.8
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "CPU使用率过高"
          description: "实例 {{ $labels.instance }} CPU使用率 {{ $value | humanizePercentage }}"
      
      - alert: HighMemoryUsage
        expr: memory_usage > 0.85
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "内存使用率过高"
          description: "实例 {{ $labels.instance }} 内存使用率 {{ $value | humanizePercentage }}"
      
      - alert: HighDiskUsage
        expr: disk_usage > 0.9
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "磁盘使用率过高"
          description: "实例 {{ $labels.instance }} 磁盘使用率 {{ $value | humanizePercentage }}"

  - name: application_alerts
    rules:
      - alert: HighErrorRate
        expr: rate(http_requests_error_total[5m]) / rate(http_requests_total[5m]) > 0.01
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "错误率过高"
          description: "服务 {{ $labels.service }} 错误率 {{ $value | humanizePercentage }}"
      
      - alert: HighLatency
        expr: histogram_quantile(0.99, rate(http_request_duration_seconds_bucket[5m])) > 0.5
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "P99延迟过高"
          description: "服务 {{ $labels.service }} P99延迟 {{ $value }}s"
```

## 三、ELK（日志监控）

### 日志分类

| 日志类型 | 说明 | 示例 |
|----------|------|------|
| 访问日志 | HTTP请求日志 | GET /api/orders 200 50ms |
| 应用日志 | 业务逻辑日志 | 订单创建成功，ID: 123456 |
| 错误日志 | 异常日志 | NullPointerException at ... |
| 慢查询日志 | 慢SQL日志 | SELECT ... 耗时 1.5s |

### Filebeat 配置

```yaml
filebeat.inputs:
  - type: log
    enabled: true
    paths:
      - /var/log/app/*.log
    fields:
      service: order-service
      env: production

  - type: log
    enabled: true
    paths:
      - /var/log/app/error*.log
    fields:
      service: order-service
      env: production
      level: error

output.elasticsearch:
  hosts: ["elasticsearch:9200"]
  index: "logs-%{[service]}-%{+yyyy.MM.dd}"

setup.kibana:
  host: "kibana:5601"
```

### 日志格式（JSON）

```json
{
  "timestamp": "2024-01-15T10:30:00Z",
  "level": "INFO",
  "service": "order-service",
  "traceId": "abc123",
  "userId": "user_001",
  "orderId": "123456",
  "message": "订单创建成功",
  "duration": 50
}
```

### Kibana 常用查询

```
# 查询错误日志
level: ERROR

# 查询某个订单的所有日志
orderId: 123456

# 查询某个服务的慢请求
service: order-service AND duration > 100

# 查询某个traceId的全链路日志
traceId: abc123
```

## 四、SkyWalking（链路追踪）

### 集成 SkyWalking

**Java 应用：**
```java
# 启动参数
-javaagent:/path/to/skywalking-agent.jar
-Dskywalking.agent.service_name=order-service
-Dskywalking.collector.backend_service=localhost:11800
```

**Go 应用：**
```go
import (
    "github.com/SkyAPM/go2sky"
    "github.com/SkyAPM/go2sky/reporter"
)

func main() {
    r, _ := reporter.NewGRPCReporter("localhost:11800")
    tracer, _ := go2sky.NewTracer("delivery-service", go2sky.WithReporter(r))
    // ...
}
```

### 链路追踪查看

**场景：** 订单创建慢

1. 打开 SkyWalking UI (http://localhost:8080)
2. 选择服务 order-service
3. 查看慢请求的 Trace ID
4. 点击 Trace 查看完整链路
5. 定位哪个环节慢（DB查询？MQ发送？）

## 六、健康检查

### HTTP 健康检查

```java
@RestController
@RequestMapping("/actuator")
public class HealthController {
    
    @GetMapping("/health")
    public Map<String, Object> health() {
        Map<String, Object> result = new HashMap<>();
        result.put("status", "UP");
        result.put("database", checkDatabase());
        result.put("redis", checkRedis());
        result.put("mq", checkMQ());
        return result;
    }
}
```

### Kubernetes 存活/就绪探针

```yaml
livenessProbe:
  httpGet:
    path: /actuator/health
    port: 8080
  initialDelaySeconds: 60
  periodSeconds: 10
  timeoutSeconds: 5
  failureThreshold: 3

readinessProbe:
  httpGet:
    path: /actuator/health
    port: 8080
  initialDelaySeconds: 30
  periodSeconds: 10
  timeoutSeconds: 5
  failureThreshold: 3
```
