# 系统测试与评估

## 3.1 测试方法

### 3.1.1 功能测试

功能测试主要测试系统的各项功能是否正常运行，包括：

1. **用户管理功能测试**：测试用户注册、登录、个人信息管理等功能。
2. **盲盒管理功能测试**：测试盲盒展示、分类、详情查看、购买等功能。
3. **订单管理功能测试**：测试订单创建、查询、状态更新等功能。
4. **配送管理功能测试**：测试配送员注册、订单分配、配送追踪等功能。
5. **系统管理功能测试**：测试后台管理、数据统计、系统设置等功能。

### 3.1.2 性能测试

性能测试主要测试系统在不同负载下的表现，包括：

1. **响应时间测试**：测试系统在不同操作下的响应时间。
2. **并发测试**：测试系统在多用户同时操作下的表现。
3. **稳定性测试**：测试系统在长时间运行下的稳定性。

### 3.1.3 安全性测试

安全性测试主要测试系统的安全性能，包括：

1. **数据加密测试**：测试用户敏感信息是否进行了加密存储。
2. **权限控制测试**：测试不同角色的权限是否正确设置。
3. **防攻击测试**：测试系统是否能够抵御常见的网络攻击。

### 3.1.4 用户体验测试

用户体验测试主要测试系统的用户友好性，包括：

1. **界面测试**：测试系统界面是否美观、易用。
2. **操作测试**：测试系统操作是否简单、直观。
3. **满意度调查**：通过问卷调查用户对系统的满意度。

## 3.5 问题与改进

### 3.5.1 存在的问题

1. **配送范围限制**：当前系统的配送范围仅限于校园内，无法覆盖校外区域。
2. **支付方式单一**：当前系统仅支持微信支付，未支持其他支付方式。
3. **盲盒种类有限**：当前系统的盲盒种类相对较少，需要进一步丰富。
4. **配送员管理**：配送员的管理机制需要进一步完善，包括评分体系和奖惩机制。

### 3.5.2 改进措施

1. **扩展配送范围**：考虑将配送范围扩展到校园周边区域，满足更多用户的需求。
2. **增加支付方式**：增加支付宝、银行卡等支付方式，提高支付的便捷性。
3. **丰富盲盒种类**：与更多供应商合作，增加盲盒的种类和数量。
4. **完善配送员管理**：建立完善的配送员评分体系和奖惩机制，提高配送服务质量。

## 3.6 评估结论

通过系统测试与评估，我们可以得出以下结论：

1. **系统功能完备**：系统实现了用户管理、盲盒管理、订单管理、配送管理和系统管理等核心功能，满足了校园用户的需求。
2. **系统性能良好**：系统在响应时间、并发用户数和稳定性方面均表现良好，能够满足校园场景的使用需求。
3. **系统安全可靠**：系统在数据加密、权限控制和防攻击方面均符合安全要求，能够保护用户的信息安全。
4. **用户体验良好**：系统在界面设计、操作便捷性、功能完整性和系统稳定性方面均得到了用户的认可，整体满意度较高。
